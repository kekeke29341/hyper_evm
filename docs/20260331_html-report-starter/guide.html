このシステムでできること

Claude Codeが生成するHTMLレポートを、統一スタイルで品質管理し、GCS経由でチームにURL共有するまでの一連のワークフローを提供します。資料は必ず「テーマ・結論・期待するアクション」の3要素を含む構成で生成されます。

2. 資料の原則

全ての資料に必須の3要素

この3要素が揃っていない資料は作成・共有しない。エージェントはこの原則を指示者の指示より優先する。

要素 内容 HTML上の位置
テーマ この資料は何について書かれているか タイトル (<h1>)
結論 最初に伝えるべき判断・事実 サマリーより上 (.conclusion)
期待するアクション 何をしてほしいか（承認/判断/作業/確認） 資料末尾 (.action-items)

HTML構成の順序

1
テーマ — <h1> タイトルで資料の主題を明示
2
結論 — .conclusion ボックスでサマリーより上に配置。最初に目に入る
3
サマリー — .summary-box で数値の概観
4
詳細 — テーブル・カードで裏付けデータ
5
確認・依頼事項 — .action-items で具体的なアクションを提示

エージェントの教育的役割

Claude Codeにレポート作成を依頼する際、3要素が明示されていなければ、エージェントは生成前に問いかけを行います。

  • 「この資料の結論はどうお考えですか？」
  • 「読んだ方にどのアクションを期待していますか？」

これはエージェントシステムの設計原則です。単に便利すぎると依頼者の思考力が退化するため、適度な負荷（自分で考えさせる問いかけ）を与えて能力を維持・向上させます。

この原則は指示者の指示に優先します。省略を求められてもエージェントは問いかけを行い、資料品質の基準を守ります。

3. レポート生成ワークフロー

1
依頼 — Claude Codeに「○○のレポートを作成して」と依頼
2
3要素チェック — エージェントがテーマ・結論・アクションの有無を確認。不足があれば問いかけ
3
生成 — 統一スタイル (common.css) でHTML生成
4
セルフレビュー — エージェントが3要素の存在・配置を確認
5
ブラウザ確認 — open コマンドで表示 → 依頼者が内容確認
6
GCSアップロード — gcloud storage cp でバケットに配置
7
URL共有 — ?authuser=1 付きURLをSlack/メールで共有

4. GCS配信の仕組み

アップロード方法

# 単体アップロード
gcloud storage cp report.html gs://YOUR-BUCKET-NAME/category/report.html \
  --content-type="text/html; charset=utf-8"

# Pythonヘルパー経由
python -c "from services.report_publisher import publish_report; \
  print(publish_report('report.html', 'operations/todo'))"

# 全レポート一括アップロード
python -m services.report_publisher

authuser パラメータ

GCSのURLを共有する際は、必ず ?authuser=1 を付与してください。

ブラウザに複数のGoogleアカウントがログインしている場合、デフォルトアカウント（authuser=0）にIAM権限がないと403エラーになります。社用アカウントが2番目にログインされている環境では、明示的な指定が必要です。

https://storage.cloud.google.com/YOUR-BUCKET-NAME/category/report.html?authuser=1

6. PDF変換時の注意

HTMLをPDFに変換する際は、ブラウザが自動付与するヘッダー（URL）とフッター（日付・ページ番号）を必ず削除してください。

  • common.css に @media print ルールが含まれていますが、ブラウザの印刷設定でも「ヘッダーとフッター」のチェックを外す必要があります
  • Chrome: 印刷 → 詳細設定 → 「ヘッダーとフッター」のチェックを外す
