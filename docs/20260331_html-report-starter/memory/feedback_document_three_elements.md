name: 資料の3要素とエージェントの教育的役割
description: 全資料にテーマ・結論・期待するアクションを必須化。依頼者に考えさせてから生成する（人を退化させない原則）
type: feedback
---

資料・レポートには必ず以下の3要素を含めること:

1. **テーマ** — この資料は何について書かれているか
2. **結論** — 読者に最初に伝えるべき判断・事実（サマリーより上に配置）
3. **期待するアクション** — 読者に何をしてほしいか（資料末尾に「確認・依頼事項」セクション）

**Why:** 資料の目的は読者に行動を促すこと。テーマ・結論・アクションが欠けた資料は「情報の羅列」であり、読者に判断コストを転嫁している。また、エージェントが全てを補完すると依頼者の思考力が退化する。適度な負荷（自分で考えさせる問いかけ）を与えることで能力を維持させるのがエージェントシステムの原則。

**How to apply:**
- レポート生成を依頼されたら、3要素が揃っているか確認。不足があれば**生成前に**依頼者に問いかける
- 問いかけ例: 「この資料の結論はどうお考えですか？」「読者にどのアクションを期待していますか？」
- エージェントが勝手に結論やアクションを補完しない。依頼者自身に考えさせる
- HTML構成: テーマ(h1) → 結論(.conclusion) → サマリー → 詳細 → 確認・依頼事項(.action-items)
- PDF変換時はブラウザ自動付与のヘッダー・フッター（URL・日付・ページ番号）を削除
