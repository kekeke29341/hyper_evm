人間への情報提供は統一されたフォーマットのHTMLで生成する。資料作成の3原則:

1. **同じ資料は同じ形式で** — 同種の資料は毎回同じHTML構造で出力する
2. **同じ目的なら情報源が変わっても共通化** — 情報源（API、DB、ファイル等）が異なっても、「チームに情報を提示する」という目的が同じなら同じフォーマットを使う
3. **同じ概念は同じように表現する** — ステータス、タグの色・表記を全HTMLで統一する

**Why:** フォーマットがバラバラだと、読み手が毎回「これは何を見ればいいのか」を考える認知コストが発生する。統一されたフォーマットにより、内容の判断に集中できる。

**How to apply:**
- 新しいレポートHTMLを作る際は、既存HTMLの構造・色・UIパターンを踏襲する
- 共通する要素（ステータスラベル、展開UI、ハイライト色）は同じCSSクラス・同じ配色を使う
