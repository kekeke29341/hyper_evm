Hyperpool Operation Map

想定されるユーザーの動きと、運営の動き

Hyperpool を公開した後に起きる行動を、ユーザー・運営・システムの3視点で並べた1枚資料です。 打ち合わせでは「ユーザーに何を案内するか」「運営が毎日何を確認するか」「本番前に何を決めるか」をこの順で確認してください。

ユーザー還元 67% 運営取り分 33% Cashdrop JST 7:00 自動送金 LP レンジ +10% / −30% 対象: WHYPE/USDC 0.05%

まず押さえる数字

ユーザー説明・運営合意の前提

67% Project X LP 手数料からユーザーへ日次 USDC 還元する比率
33% 運営ウォレットへ送る比率。公開後に変えにくい前提値
7:00 毎朝 JST の運営処理で対象ユーザーへUSDCを自動送金
+10 / −30% keeper が維持する固定レンジ。ユーザーごとの選択はなし
説明上の注意: APR は保証値ではなく参考値です。ユーザーには「Project X 参考 APY × 67% が還元目安」と説明し、価格変動・impermanent loss・スマートコントラクトリスクを別途明記します。

ユーザーの動き

初回来訪から継続利用まで

初回ユーザー

目的は「USDC または HYPE を預け、Project X LP 運用を任せる」ことです。

1
サイト訪問・ウォレット接続
HyperEVM Testnet 998 または Mainnet 999 に接続。チェーン違いはネットワーク切替バナーで案内。
2
資金準備
HyperEVM 上の USDC が推奨。他チェーン資金は Mainnet 999 で Li.FI Bridge を利用。Testnet 998 では Bridge は実質非対応。
3
Position で deposit
USDC または HYPE を Vault に預ける。ユーザーは Vault share を受け取り、LP NFT は ProjectXAdapter が管理。
4
Dashboard / Position を確認
シェア価値、預入状態、レンジ、過去の自動送金履歴を確認。APR は参考表示として扱う。
5
毎朝 Cashdrop を自動受取
JST 7:00 の運営処理で、対象額がユーザーウォレットへUSDC送金される。ユーザーの署名・claim操作は不要。
6
必要に応じて withdraw
Vault share を burn し、比例配分で USDC + WHYPE を受け取る。ガス用 HYPE は常に必要。

継続ユーザー・紹介ユーザー

毎日見る場所と、問い合わせにつながりやすいポイントです。

Cashdrop 自動送金 紹介コード 追加 deposit 送金Tx確認 チェーン違い
ユーザー行動 よくある期待・質問 案内すること
毎朝の入金を確認 「今日はいくらもらえるか」 7:00 以降にウォレット・Cashdropタブ・Dashboardを確認。対象者にはUSDCが自動送金される。
紹介リンクを共有 「紹介報酬は別枠か」 紹介者 15%・被紹介者 +10% は 67% プール内での再配分。総額が増える保証ではない。
APR を見る 「この利回りが保証されるか」 保証 APR ではなく参考値。手数料量、価格変動、レンジ外時間で変わる。
出金する 「USDC だけで返るか」 Vault の構成比に応じて USDC + WHYPE を受け取る。価格変動リスクあり。

運営の動き

公開前・日次・緊急時

公開前にやること

A
数値と説明文の合意
33% / 67%、Cashdrop 時刻、紹介 15% / +10%、+10% / −30% レンジ、ブリッジ手数料 0% を決定。
B
権限とウォレット整理
Vault owner、Airdrop owner、keeper、operatorWallet、緊急停止責任者を決める。Cashdropはユーザー操作なしで自動送金する。
C
インフラ確認
Vercel 本番 URL、RPC、WalletConnect、deployment JSON、cron ホスト、GitHub Actions / VPS 移行方針を確認。
D
ゲストデモ（完了）
2026-06 に削除済み。未接続時は空状態とウォレット接続促進のみ。
