## 1. 接続方法（2 パターン）

### A. WalletConnect（PC ブラウザ ↔ 携帯ウォレット）

1. 携帯の Safari / Chrome で **https://hyper-evm-ten.vercel.app** を開く  
   （または PC で開き、WalletConnect の QR を携帯でスキャン）
2. 右上 **「ウォレット接続」**
3. **ネットワーク** で **HyperEVM Testnet (998)** を選択（初回は Testnet 推奨）
4. **WalletConnect** → 「モバイルウォレットで QR スキャン」
5. ウォレットアプリで QR をスキャン → **接続を承認**
6. チェーン追加・切替のプロンプトが出たら **998 を承認**

**期待結果:** ヘッダーに `0x0196…8ceC` のような短縮アドレスが表示される。

### B. MetaMask アプリ内ブラウザ

1. MetaMask アプリを開く
2. 下部 **ブラウザ** タブ → URL 欄に `https://hyper-evm-ten.vercel.app`
3. **ウォレット接続** → **MetaMask**（Injected）
4. ネットワーク **998** を選択して接続

**メリット:** QR 不要。MetaMask 内蔵 dApp ブラウザ向け。

## 2. テストチェックリスト

各項目で **OK / NG / メモ** を記録してください。NG 時は Explorer の tx hash を控えます。

### Phase 1 — 接続・表示

- [ ] **1-1** サイトがモバイルで横スクロールせず表示される
- [ ] **1-2** ウォレット接続モーダルが画面下から開き、3 つの選択肢（MetaMask / WalletConnect / ブラウザ）が見える
- [ ] **1-3** WalletConnect が **無効（グレーアウト）になっていない**
- [ ] **1-4** 接続後、ヘッダーに短縮アドレスが表示される
- [ ] **1-5** ネットワーク表示が **998**（Testnet）または意図した **999**（Mainnet）になっている
- [ ] **1-6** ダッシュボードの Vault 短縮アドレスが、上表の Vault と一致する

### Phase 2 — ポジション（Vault 入出金）

**タブ: ポジション** (`/position`)

- [ ] **2-1** ウォレット残高（HYPE / USDC）が表示される
- [ ] **2-2** Vault が **Live** バッジ付きで表示される
- [ ] **2-3** **預入 (Deposit)** → モーダルで **HYPE** または **USDC** を選択
- [ ] **2-4** 少額（例: HYPE `0.001` または USDC `0.01`）を入力 → **預入** → ウォレットで **Approve**（USDC 時）→ **Confirm**
- [ ] **2-5** Explorer で `depositHYPE` / `depositUSDC` tx が **Success**
- [ ] **2-6** Vault シェア・残高が更新される（数秒〜30 秒待つ）
- [ ] **2-7** **引出 (Withdraw)** → シェアの 1〜5% → 署名 → ウォレットに HYPE/USDC が戻る

**Testnet 参考:** CLI `testnet-vault-smoke.mjs` で withdraw + re-deposit は 2026-06-26 時点で成功済み。

### Phase 3 — Cashdrop

**タブ: Cashdrop** (`/cashdrop`)

- [ ] **3-1** 接続済みで「次回配布まで」のカウントダウンが表示される
- [ ] **3-2** 配布対象がある場合: 自動送金済み USDC 額が表示される（手動 Claim ボタンは **なし** — 自動送金モデル）
- [ ] **3-3** 対象外の場合: 空状態メッセージが表示される（エラーではない）

**注意:** Testnet では merkle エントリが **EXPIRED** の場合、旧 CLI claim は失敗します。UI は自動送金結果の **表示確認** が主目的です。

### Phase 4 — ダッシュボード

**タブ: ダッシュボード** (`/`)

- [ ] **4-1** 接続後、ポジション価値が `—` 以外になる（Vault 預入後）
- [ ] **4-2** 収益グラフエリアが表示される（Cashdrop 履歴はローカル storage — 初回は空でも可）

### Phase 5 — 紹介（任意）

**タブ: 紹介** (`/affiliate`)

- [ ] **5-1** 紹介コード / リンクが表示される
- [ ] **5-2** コピーボタンが動作する

### Phase 6 — Mainnet のみ（999・極小額）

Testnet がすべて OK になってから実施。

- [ ] **6-1** ネットワーク **999** で接続
- [ ] **6-2** Vault `0x2db5…502d` を確認
- [ ] **6-3** USDC `0.05` 程度の deposit → Explorer 成功
- [ ] **6-4** Position タブで Project X 実 NPM ポジション情報が表示される

## 3. トラブルシューティング

| 症状 | 対処 |
|------|------|
| WalletConnect がグレーアウト | Vercel の `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` 未設定。Reown Cloud で `*.vercel.app` を許可 |
| 接続後すぐ切断 | ウォレット側で HyperEVM (998/999) を追加・選択 |
| USDC deposit が revert | 先に USDC **approve** を Vault アドレスへ。残高不足も確認 |
| ネットワーク不一致 | モーダル内「ネットワークを選択」で 998/999 を再選択 → ウォレットで切替承認 |
| MetaMask 警告バナー | 既知（サブアカウント SC 非公開）。`WalletAlertNotice` に記載のとおり、接続自体は可能 |
| Li.FI ブリッジ失敗 (998) | **正常** — Testnet 998 は Li.FI 非対応。ブリッジタブは Mainnet 向け |

## 4. PC 側の自動テスト（携帯の代替・事前確認）

携帯テストの前に、以下を通しておくと原因切り分けが早いです。

```bash
# モバイル UI（レイアウト・タップ領域）
cd frontend && CI=true npx playwright test --project=mobile

# ウォレットモック（MetaMask 不要）
cd frontend && npm run test:e2e:wallet:mock

# Testnet on-chain 状態（MAIN 鍵）
source scripts/testnet-env.sh
cd frontend && npm run verify:testnet

# Vault 入出金スモーク（MAIN 鍵）
WITHDRAW_BPS=100 node scripts/testnet-vault-smoke.mjs

# E2E 専用鍵で Vault deposit 等（MetaMask UI 不要）
node scripts/testnet-e2e-wallet.mjs
```

`.env.testnet` に以下が必要（Synpress / CLI E2E 共通）:

```bash
E2E_PRIVATE_KEY=0x...          # MAIN_PRIVATE_KEY と別の低価値鍵
E2E_ADDRESS=0x...
SYNPRESS_WALLET_PASSWORD=...     # Synpress キャッシュ構築時のみ
```

### Synpress（PC + MetaMask 拡張・実 tx）

**MetaMask ポップアップは GUI 環境が必要**です。Cursor 内のヘッドレス実行では `connectToDapp` が失敗することがあります。

```bash
source scripts/testnet-env.sh
cd frontend && npm run test:e2e:synpress:cache   # 初回
HEADLESS=false RUN_TESTNET_E2E=true npm run test:e2e:wallet:testnet   # Mac 画面あり推奨
```
