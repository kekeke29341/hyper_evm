# Testnet 運用（Chain 998）

## 結論

**HyperEVM Testnet への Hyperpool デプロイは完了しています。**  
998 では本物の Project X NPM の代わりに **`MockProjectXNPM`** を使い、Vault 経由の代理 LP・keeper リバランス・Cashdrop を E2E 検証できます。

運営が継続的にやること:

1. **Keeper リバランス**（`keeper-rebalance.mjs` — cron 推奨）
2. **日次報酬**（`daily-rewards.mjs` または testnet 用 smoke）
3. **`vaultShareHolders` スナップショット**（daily-rewards 前に必須）
4. **フロント Vercel** — `998.json` 変更後は Redeploy

## 環境準備

`.env.testnet` に運営ウォレットを設定（**commit しない**）:

```bash
MAIN_PRIVATE_KEY=...
MAIN_ADDRESS=0x...
```

シェルで読み込み:

```bash
source scripts/testnet-env.sh
```

---

## 一括 E2E（推奨）

```bash
source scripts/testnet-env.sh
./scripts/testnet-run-all.sh
```

実行内容: 株主 sync → verify → rebalance → mock 手数料 accrue → harvest → daily smoke → claim → vault smoke → 再 sync → verify

環境変数:

| 変数 | デフォルト | 説明 |
|------|-----------|------|
| `POOL_USDC` | `0.01` | daily-rewards-smoke の分配プール |
| `WITHDRAW_BPS` | `500` | vault smoke の出金率（bps） |

### 初回セットアップ（Vault 未使用時）

```bash
source scripts/testnet-env.sh
# spot USDC が少ない場合: LP_USDC=0 VAULT_DEPOSIT_HYPE=0.02
node scripts/testnet-post-deploy.mjs
node scripts/testnet-sync-shareholders.mjs
node scripts/sync-abi.mjs
```

### Keeper（cron 例）

```bash
# 6 時間ごと
0 */6 * * * cd /path/hyper_evm && source scripts/testnet-env.sh && DEPLOYMENT_CHAIN=998 SKIP_ORACLE=1 node scripts/keeper-rebalance.mjs
```

`SKIP_ORACLE=1` — Mock NPM testnet では oracle 偏差チェックをスキップ可能。

### 日次報酬（cron 例 · JST 7:00）

```bash
0 7 * * * TZ=Asia/Tokyo cd /path/hyper_evm && source scripts/testnet-env.sh && DEPLOYMENT_CHAIN=998 node scripts/testnet-sync-shareholders.mjs && DEPLOYMENT_CHAIN=998 node scripts/daily-rewards.mjs
```

**998 の制約:** 本物 USDC 手数料が harvest できないため、`daily-rewards.mjs` で `pendingUserRewards = 0` になることがある。その場合:

```bash
POOL_USDC=0.01 node scripts/testnet-daily-rewards-smoke.mjs
node scripts/testnet-wallet-actions.mjs
```

## Testnet でできないこと

| 項目 | 理由 |
|------|------|
| 本物 Project X NPM | 998 には未デプロイ → Mock NPM を使用 |
| USDC 手数料 harvest 完全 E2E | **現行 998 NPM は旧 bytecode** — 再デプロイ後は approve + accrue で可能 |
| Li.FI bridge | chain 998 非対応 |

本番相当の USDC 手数料ループは **mainnet (999)**、**Anvil MockERC20**、または **998 再デプロイ後** で検証。

Cron 例: `scripts/cron/hyperpool.example.crontab`

## 再デプロイが必要になるケース

- コントラクトロジック変更で **新アドレス** が必要
- owner / keeper 移管後に旧 Vault を使い続けられない

通常の運用（リバランス・Cashdrop・入出金）では **再デプロイ不要**。
