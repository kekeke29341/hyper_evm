# Testnet 運用（Chain 998）

## 結論

**HyperEVM Testnet へのコントラクトデプロイは完了しています。**  
運営が追加でやるべきは主に次の 2 点です。

1. **プールへの流動性投入**（Swap を実際に動かす）
2. **（任意）Cashdrop の設定**（Merkle + USDC fund + フロント JSON 同期）

フロント（Vercel）は別途、環境変数が Testnet 向けか確認してください。

## 運営作業 B: 流動性シード（Swap 必須）

### 方法 1: 一括スクリプト（推奨）

`.env.testnet` に `MAIN_PRIVATE_KEY`（運営ウォレット）を設定:

```bash
node scripts/testnet-post-deploy.mjs
```

実行内容:

1. Hyperliquid Testnet から **USDC を EVM にブリッジ**
2. **HYPE → kHYPE** に wrap
3. Router 経由で **addLiquidity**（プールに kHYPE + USDC）
4. Merkle root 設定 + Airdrop へ USDC `fund`（残高がある場合）
5. `998.json` に `airdropEntries` を書き込み

### 方法 2: デプロイスクリプト再実行（流動性のみ）

運営ウォレットに十分な kHYPE / USDC がある場合:

```bash
export PRIVATE_KEY=0x...
SEED_LIQUIDITY=true \
SEED_KHYPE=1000000000000000000 \
SEED_USDC=2000000000 \
./scripts/deploy-testnet.sh
```

（コントラクトは再デプロイされず、シード部分のみ broadcast される想定ではなく、スクリプト全体が走る点に注意。新規デプロイが不要なら **方法 1** を優先。）

### 方法 3: GUI + MetaMask

運営ウォレットでアプリの **Position** タブから LP 追加（トークン保有が前提）。

## 運営作業 C: ABI / フロント同期

流動性・Cashdrop 設定後:

```bash
node scripts/sync-abi.mjs
git add contracts/deployments/998.json frontend/src/lib/contracts/deployments/998.json
git commit -m "Update testnet deployment state"
git push
```

Vercel が自動再デプロイされ、ユーザー向け UI に最新アドレス・`airdropEntries` が反映されます。

---

## 運営作業 D: Cashdrop（任意）

Admin（`/admin`、`NEXT_PUBLIC_ADMIN_ENABLED=true` のビルド）または CLI:

1. 対象者 CSV（`address,amount` — USDC 6 decimals）
2. `setMerkleRoot` + 期限
3. MerkleAirdrop へ USDC `approve` + `fund`
4. `airdropEntries` を `998.json` に載せて sync-abi

詳細: [admin-guide.md](../admin-guide.md)

| 条件 | 説明 |

**Li.FI クロスチェーンは Testnet では使えません。** 同一チェーン Swap で検証してください。

---

## 再デプロイが必要になるケース

- コントラクトロジックを変更し **新アドレス** で出し直す
- `feeToSetter` / owner を移管したうえで Factory 設定を変える
- 誤デプロイでアドレスが無効

通常の「Swap を動かしたい」だけなら **再デプロイではなく流動性シード** で足ります。
