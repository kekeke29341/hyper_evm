# 本番環境運用（Chain 999 / Mainnet）

本番は **HyperEVM Mainnet (Chain ID 999)** 向けです。Testnet (998) で Vault E2E を検証してから進めてください。

---

## Testnet との違い

| 項目 | Testnet (998) | Mainnet (999) |
|------|---------------|---------------|
| 現状 | **Hyperpool デプロイ済み**（Mock NPM） | 未デプロイ想定 |
| Project X NPM | MockProjectXNPM | 本物 NPM `0xeaD19…` |
| フロント `DEFAULT_CHAIN_ID` | `998` | **`999`** |
| Li.FI クロスチェーン | **非対応** | 対応 |
| USDC 手数料 harvest | 不完全（FiatToken mint 不可） | **本番ループ可能** |
| 資金 | テストトークン | **実資産** |

---

## 本番公開の流れ（概要）

### 1. コントラクトデプロイ

```bash
source scripts/testnet-env.sh   # または .env.mainnet
./scripts/deploy-mainnet.sh
node scripts/sync-abi.mjs
```

- big blocks 有効化、十分な HYPE ガス
- `contracts/deployments/999.json` をコミット

### 2. Vault 初回 deposit + LP

Testnet の `testnet-post-deploy.mjs` 相当を mainnet 向けに実行:

- Vault へ USDC / HYPE deposit → Project X NPM 上に LP mint
- 本番では金額・価格・監査済みウォレットを使用

### 3. Keeper / 日次 cron

```bash
DEPLOYMENT_CHAIN=999 node scripts/keeper-rebalance.mjs
DEPLOYMENT_CHAIN=999 node scripts/testnet-sync-shareholders.mjs  # mainnet 用に rename 可
DEPLOYMENT_CHAIN=999 node scripts/daily-rewards.mjs
```

### 4. フロント（Vercel）

| Variable | 本番値 |
|----------|--------|
| `NEXT_PUBLIC_DEFAULT_CHAIN_ID` | **`999`** |
| `NEXT_PUBLIC_MAINNET_RPC` | `https://rpc.hyperliquid.xyz/evm` |
| `NEXT_PUBLIC_ADMIN_ENABLED` | **`false`** |

### 5. セキュリティ

- [security-review-2026-06-15.md](../security-review-2026-06-15.md) 確認
- owner / keeper ウォレット: マルチシグ推奨
- `PRIVATE_KEY` は Vercel・GitHub に入れない
