# 本番環境運用（Chain 999 / Mainnet）

本番は **HyperEVM Mainnet (Chain ID 999)** 向けです。Testnet で手順を検証してから進めてください。

---

## Testnet との違い

| 項目 | Testnet (998) | Mainnet (999) |
|------|---------------|---------------|
| 現状 | **コントラクトデプロイ済み** | `999.json` はプレースホルダー or 未デプロイ想定 |
| フロント `DEFAULT_CHAIN_ID` | `998` | **`999`** |
| Li.FI クロスチェーン | **非対応** | 対応（UI 設計上） |
| トークン | テスト用 kHYPE / USDC | 本番アドレス（`HyperCoreConstants`） |
| 資金 | テストトークン | **実資産** — 運用・セキュリティ厳格化 |

---

## 本番公開の流れ（概要）

### 1. コントラクトデプロイ

```bash
export PRIVATE_KEY=0x...   # 本番運用ウォレット（厳重管理）
./scripts/deploy-mainnet.sh
node scripts/sync-abi.mjs
```

- デプロイ前: HyperEVM で **big blocks** 有効化
- ガス: 本番 HYPE を十分に用意
- デプロイ後: `contracts/deployments/999.json` をコミット

### 2. 初期流動性（Swap 必須）

Testnet と同様、**Pair プールに kHYPE + USDC を LP として投入**が必要です。

- 運営 or パートナーが `addLiquidity`
- 本番では金額・価格インパクト・監査済みウォレットを使用

Mainnet 向けの `testnet-post-deploy.mjs` 相当は **本番 API・資金に合わせて手動 or スクリプト新設** が必要です（現状は Testnet 用スクリプトが中心）。

### 3. フロント（Vercel）

| Variable | 本番値 |
|----------|--------|
| `NEXT_PUBLIC_DEFAULT_CHAIN_ID` | **`999`** |
| `NEXT_PUBLIC_MAINNET_RPC` | `https://rpc.hyperliquid.xyz/evm` |
| `NEXT_PUBLIC_ADMIN_ENABLED` | **`false`**（Production） |
| WalletConnect | 本番ドメインを Allowed Domains に追加 |

### 4. Cashdrop / ポイント

- Merkle リスト・配布額は本番資金に合わせて Admin または CLI
- ポイント: デプロイ時 `authorizePool` — 追加プールは Admin Pools タブ

### 5. セキュリティ

- [security-review-2026-06-12.md](../security-review-2026-06-12.md) の未対応項目を確認
- owner / `feeToSetter` ウォレット: マルチシグ or ハードウェア推奨
- `PRIVATE_KEY` は Vercel・GitHub に **絶対に入れない**

## 本番で「コントラクトに入れる」もの（再掲）

| 目的 | 何をどこに |
|------|-----------|
| Swap | Pair プールへ kHYPE + USDC（`addLiquidity`） |
| Cashdrop | MerkleAirdrop へ USDC（`fund`） |
| ガス | 運営ウォレットに HYPE |
| Points | コントラクトへの PTS 預け入れは **不要**（手数料で記録） |
