# 現状と全体像

運営が環境を立ち上げるとき、**何がどこで動くか**を最初に押さえてください。

---

## 2 層構造

Hyperpool の「稼働」は **フロント** と **オンチェーン** の両方が揃って初めてユーザー体験が完成します。

```
┌──────────────────────────────────────────────────────────────┐
│ レイヤー 1: フロントエンド（Vercel / localhost）              │
│ ・ブラウザ + MetaMask から操作する GUI                        │
│ ・Deposit（Li.FI）/ Position（Vault）/ Cashdrop / Dashboard  │
│ ・設定: NEXT_PUBLIC_DEFAULT_CHAIN_ID, RPC, WalletConnect 等   │
└────────────────────────────┬─────────────────────────────────┘
                             │ JSON-RPC（読取・TX 送信）
┌────────────────────────────▼─────────────────────────────────┐
│ レイヤー 2: HyperEVM スマートコントラクト                     │
│ ・ Chain 998 = Testnet / Chain 999 = Mainnet                  │
│ ・ HyperpoolVault → ProjectXAdapter → Project X NPM           │
│ ・ Keeper rebalance (+10% / −30%) / daily harvest → Cashdrop  │
└──────────────────────────────────────────────────────────────┘
```

**Vercel だけデプロイしても Vault LP は動きません。** ユーザーの **Vault deposit** と運営の **keeper 運用** が揃って初めて Project X 代理 LP が稼働します。

---

## ユーザー体験の流れ

```
ユーザー → Vault deposit (USDC / HYPE)
         → Adapter が Project X NPM 上に LP mint
Keeper   → 価格 drift で rebalance (+10% / −30%)
Daily    → harvestFees (USDC 手数料 33% 運営 / 67% 自動送金)
         → JST 7:00 Cashdrop payout
Dashboard → 自動送金履歴を可視化
```

---

## 機能ごとに運営がやること（一覧）

| 機能 | コントラクトに資金を「入れる」？ | 運営の作業 |
|------|----------------------------------|------------|
| **画面表示** | 不要 | Vercel デプロイ・env 設定 |
| **Vault LP** | **ユーザーが Vault へ USDC / HYPE deposit** | GUI。Testnet smoke のみ `testnet-post-deploy.mjs` |
| **リバランス** | 不要（keeper TX のみ） | `keeper-rebalance.mjs`（cron 推奨） |
| **日次報酬** | harvest 後 Airdrop 経由でUSDC自動送金 | `daily-rewards.mjs` + `vaultShareHolders` sync |
| **Cashdrop** | 対象者へ直接USDC送金 | `distributeRewards` + `airdropEntries` / `lastCashdropDistribution` JSON 同期 |
| **Li.FI クロスチェーン** | 別経路 | **Testnet 998 は非対応**。本番 999 向け |
| **Admin** | 不要 | Preview/ローカルのみ有効、owner ウォレット |

## Testnet の現状（2026-06 · オンチェーン E2E 検証済み）

`contracts/deployments/998.json` と RPC 読取の結果:

| 項目 | 状態 |
|------|------|
| Hyperpool コントラクトデプロイ | ✅ 済み |
| MockProjectXNPM + Vault LP | ✅ tokenId あり |
| Vault 入出金 | ✅ CLI + 一部 GUI 検証 |
| Keeper リバランス | ✅ CLI 成功 |
| USDC 手数料 harvest 完全ループ | ❌ 本物 USDC mint 不可（smoke で代替） |
| Cashdrop | ✅ Merkle + claim 検証済み |
| `vaultShareHolders` | ✅ JSON 同期済み |

→ **「Vault LP + rebalance + Cashdrop」は testnet で検証済み。**  
→ **本番相当の USDC 手数料 harvest** は mainnet (999) または Anvil MockERC20 が必要。

## 運営ウォレットに常に必要なもの

| 用途 | トークン |
|------|---------|
| デプロイ・keeper TX | HYPE（ガス） |
| daily-rewards TX | HYPE（ガス） |
| 任意の事前 smoke | 少額 HYPE / USDC（本番LP原資ではない） |

Mainnet では、LP 原資はユーザーが Vault に預ける USDC / HYPE です。運営が常時 LP 資金を用意する必要はありません。

Testnet では:

- ガス: [Hyperliquid Testnet Drip](https://app.hyperliquid-testnet.xyz/drip)
- USDC / wrap / deposit smoke: `scripts/testnet-post-deploy.mjs`
