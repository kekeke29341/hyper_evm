# 検証手順（Testnet / Mainnet）

自動で済んでいることと、**ウォレットが必要な手動確認**を分けています。Mainnet は実資産を扱うため、最初は必ず極小額で実施してください。

---

## こちら（CLI）で済んでいること

- Hyperpool コントラクトデプロイ + `998.json` 更新
- Vault deposit / withdraw / re-deposit（CLI smoke）
- Keeper リバランス
- Mock 手数料 accrue + harvest（WHYPE のみ → USDC 67% pool は 0）
- Cashdrop 自動送金（daily-rewards / smoke 経由）
- `forge test` pass
- `npm test` pass
- `verify-testnet.mjs` / `npm run verify:testnet` 合格

---

## あなたがやること（推奨順）

### 1. Vercel 再デプロイ（最重要）

`998.json` はビルド時にフロントへ埋め込まれるため、**push 後に Production Redeploy が必要**です。

Vercel Dashboard → Deployments → 最新 main の **Redeploy**

確認: Position タブに正しい Vault アドレス `0xd50624c040c0b5fea3e7b8ad818924b5a2640759` が表示される。

### 2. MetaMask を Chain 998 に設定

| 項目 | 値 |
|------|-----|
| Network | HyperEVM Testnet |
| Chain ID | `998` |
| RPC | `https://rpcs.chain.link/hyperevm/testnet` |

### 3. Position タブ（Vault 入出金）

1. https://hyper-evm-ten.vercel.app を開く（または `cd frontend && npm run dev -- -p 3001`）
2. ウォレット接続
3. **Position** → 少額 USDC または HYPE を **入金**
4. Vault shares / ポジション価値が更新されること
5. （任意）**出金** で USDC + WHYPE が返ること

失敗時: ネットワークが 998 か確認。エラーに「Anvil」と出たら `NEXT_PUBLIC_DEFAULT_CHAIN_ID` 不一致。

### 4. Cashdrop

daily rewards が成功し、Airdrop から対象ウォレットへ USDC が自動送金されていること。

```bash
source scripts/testnet-env.sh
node scripts/verify-testnet.mjs
# Testnet で本物 harvest が 0 の場合は smoke で分配原資を用意:
POOL_USDC=0.01 node scripts/testnet-daily-rewards-smoke.mjs
```

JST **7:00** の daily rewards 実行で対象ウォレットへUSDCが自動送金される。送金後、`lastCashdropDistribution` / `distributionId` とダッシュボード収益履歴を確認する。

### 5. ダッシュボード

- Cashdrop 自動送金後に収益グラフが表示されること
- testnet 998 では TVL / Social Proof は非表示または `—`（想定）

オンチェーン配線:

```bash
RPC=https://rpc.hyperliquid.xyz/evm
VAULT=0x2DB5FCfC0c9Eed612A544B99C9097FbBC0Cf502d
ADAPTER=0x3193AfC8FBEEDB0c2ee5B2F9Ed287579c2aa1796

cast call $ADAPTER 'vault()(address)' --rpc-url $RPC
cast call $ADAPTER 'pool()(address)' --rpc-url $RPC
cast call $VAULT 'swapRouter()(address)' --rpc-url $RPC
cast call $AIRDROP 'vaultShareToken()(address)' --rpc-url $RPC
cast call $VAULT 'oraclePriceUsdc6PerHype18()(uint256)' --rpc-url $RPC
cast call $ADAPTER 'currentPoolPriceUsdc6PerHype18()(uint256)' --rpc-url $RPC
```

期待値:

- `adapter.vault() == VAULT`
- `adapter.pool() == 0x6c9A33E3b592C0d65B3Ba59355d5Be0d38259285`
- `vault.swapRouter() == 0x1EbDFC75FfE3ba3de61E7138a3E8706aC841Af9B`
- `airdrop.vaultShareToken() == VAULT`
- oracle価格とProject X pool価格が同じ桁で、通常は数%以内

### 2. deposit / LP range

1. 検証ウォレットに少額USDC（例: `0.07 USDC`）とガス用HYPEを用意
2. UIのPositionタブ、またはCLIから `depositUSDC(amount, receiver)`
3. `adapter.positionTokenId()` が増えること
4. `tickLower < currentTick < tickUpper` でレンジ内であること

確認コマンド:

```bash
cast call $ADAPTER 'positionTokenId()(uint256)' --rpc-url $RPC
cast call $ADAPTER 'tickLower()(int24)' --rpc-url $RPC
cast call $ADAPTER 'tickUpper()(int24)' --rpc-url $RPC
cast call 0x6c9A33E3b592C0d65B3Ba59355d5Be0d38259285 'slot0()(uint160,int24,uint16,uint16,uint16,uint8,bool)' --rpc-url $RPC
```

### 3. harvest

```bash
cast send $VAULT 'harvestFees()' --private-key $PRIVATE_KEY --rpc-url $RPC
cast call $VAULT 'pendingUserRewards()(uint256)' --rpc-url $RPC
```

注意: 極小LPかつdeposit直後は `pendingUserRewards == 0` が正常に起こり得ます。非ゼロ手数料確認は、十分なLP規模または時間経過後のProject X取引量が必要です。

### 3b. Cashdrop 自動送金（JST 7:00 を待たずに即時テスト）

`daily-rewards.mjs` は cron 用スケジュールのラッパーであり、**いつでも手動実行可能**です。Claim ボタンは不要（`distributeRewards` でウォレットへ直接 USDC 送金）。

```bash
source scripts/testnet-env.sh   # MAIN_PRIVATE_KEY = Mainnet operator

# 本番と同じフロー（harvest → pullPendingRewards → distributeRewards）
DEPLOYMENT_CHAIN=999 node scripts/daily-rewards.mjs

# RPC レート制限時は shareholder スナップショットを省略
SKIP_SYNC=1 DEPLOYMENT_CHAIN=999 node scripts/mainnet-daily-rewards-smoke.mjs
```

`pendingUserRewards == 0` のときは `POOL_USDC=0.005` を付けると運営ウォレットから分配原資を入れて **送金パスだけ** 検証できます（実 LP 手数料の確認にはなりません）。

成功条件:

- `harvestFees` tx 成功（非ゼロ手数料時は operator USDC が **即時 +33%**）
- `distributeRewards` tx 成功
- 対象ウォレット USDC 残高増加
- `999.json` の `lastCashdropDistribution` 更新 → Vercel redeploy 後 Cashdrop タブに反映

2026-06-26: Mainnet で `harvestFees` → `+0.000016 USDC`（運営33%）→ `distributeRewards` → `+0.000034 USDC`（ユーザー67%）を即時確認。

### 4. rebalance

`refPrice` は Project X pool価格またはその近傍を使用します。VaultはHyperCore oracle価格との乖離が `maxRebalanceDeviationBps`（標準5%）を超えるとrevertします。

```bash
cast call $ADAPTER 'currentPoolPriceUsdc6PerHype18()(uint256)' --rpc-url $RPC
cast send $VAULT 'rebalance(uint256)' <refPrice> --private-key $PRIVATE_KEY --rpc-url $RPC
```

成功条件:

- tx成功
- `positionTokenId` が更新される
- rebalance後も `tickLower < currentTick < tickUpper`
- `totalAssetsUsdc()` が大きく毀損していない

2026-06-25 smokeでは `0.07 USDC` deposit、`harvestFees`、Project X価格 +0.5% の `rebalance` が成功しました。

## 資金が足りないとき

| 症状 | 対処 |
|------|------|
| post-deploy で spotSend 失敗 | `node scripts/testnet-bootstrap.mjs` または `LP_USDC=0 VAULT_DEPOSIT_HYPE=0.02` |
| daily-rewards-smoke で USDC 不足 | spot USDC を増やすか `POOL_USDC` を下げる |
| Cashdrop 自動送金失敗 | daily log、Airdrop残高、distributionId、対象者リストを確認 |
| harvest 後 rewards = 0 | 998 制約（USDC mint 不可）— smoke で代替 |
