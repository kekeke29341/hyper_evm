# Testnet 検証手順（あなたがやること）

自動で済んでいることと、**ウォレットが必要な手動確認**を分けています。

---

## こちら（リポジトリ / オンチェーン）で済んでいること

- コントラクトデプロイ + `998.json` 更新
- `forge test` 73 件パス（ローカル）
- `node scripts/verify-testnet.mjs` による RPC スモークチェック
- 最小流動性シード（現状: 約 0.03 kHYPE + 0.1 USDC）

---

## あなたがやること（推奨順）

### 1. Vercel 再デプロイ（最重要）

`998.json` はビルド時にフロントへ埋め込まれるため、**push 後に Production Redeploy が必要**です。

Vercel Dashboard → プロジェクト → Deployments → 最新 main の **Redeploy**

確認: 再デプロイ後、スワップに `0.01` kHYPE を入力すると USDC 見積もりが `—` でなく数値になる。

### 2. MetaMask を Chain 998 に設定

| 項目 | 値 |
|------|-----|
| Network | HyperEVM Testnet |
| Chain ID | `998` |
| RPC | `https://rpcs.chain.link/hyperevm/testnet` |

アプリ上部に **ネットワーク切替バナー** が出たら「HyperEVM Testnet (998) に切替」を押す。

### 3. スワップ TX（少額）

1. https://hyper-evm-ten.vercel.app を開く
2. ウォレット接続（デプロイウォレット or テスト用ウォレット）
3. kHYPE 残高があること（Faucet: https://app.hyperliquid-testnet.xyz/drip）
4. **0.01 kHYPE → USDC** など少額スワップ

失敗時: バナーで 998 か確認。エラーが「Anvil」ならネットワーク不一致の可能性大。

### 4. Cashdrop（任意）

現状、Merkle は **100 USDC 請求**設定だがコントラクト残高が不足している場合があります。

```bash
# spot / EVM に USDC を足してから
node scripts/testnet-fund-airdrop.mjs
```

`node scripts/verify-testnet.mjs` の「Airdrop USDC >= claim total」が ✓ になってから請求。

### 5. ポイントタブ

- クラッシュしないこと
- スワップ後、エポック内は **プレビュー分**、エポック終了後は **claim 済み分** が増えること

## ローカル検証コマンド

```bash
node scripts/verify-testnet.mjs
cd contracts && forge test
cd frontend && npm run typecheck && npm run build
```

---

## 資金が足りないとき

| 症状 | 対処 |
|------|------|
| post-deploy で spotSend 失敗 | `node scripts/testnet-bootstrap.mjs`（drip）または `BRIDGE_USDC` を下げる |
| 流動性が極小でレートが不安定 | spot USDC を増やして `BRIDGE_USDC=6 LP_USDC=5` で post-deploy 再実行 |
| Cashdrop 請求失敗 | `node scripts/testnet-fund-airdrop.mjs` |
