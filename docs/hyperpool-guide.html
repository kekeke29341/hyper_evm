Level 1

初心者向け — ゼロから使う

Hyperpool とは？

Hyperpool は HyperEVM（Hyperliquid の EVM 互換チェーン）上で動く AMM 型 DEX です。中央の取引所ではなく、スマートコントラクトと流動性プールを通じて、ユーザー同士がトークンを交換します。

用語意味
DEX分散型取引所。ウォレットから直接取引する
AMM自動マーケットメーカー。プール内の残高比率で価格が決まる
HyperEVMHyperliquid エコシステムの EVM レイヤー（Chain ID 998=Testnet, 999=Mainnet）
kHYPE / USDC主な取引ペア。kHYPE ↔ USDC のスワップ・LP が可能
ウォレットMetaMask 等。トランザクションの署名・送信に必要

はじめ方（5ステップ）

1
ウォレットを用意 — MetaMask 等をインストールし、シードフレーズを安全に保管
2
HyperEVM ネットワークを追加 — Testnet（998）または Mainnet（999）をウォレットに登録
3
Hyperpool を開く — Vercel 上のフロントエンド URL にアクセス
4
Connect Wallet — 右上からウォレットを接続
5
Swap タブで小さな量を試す — スリッページ設定（設定画面）を確認してから取引
ウォレット未接続でもできること

タブの閲覧、見積もり表示（プールに流動性がある場合）などは可能です。スワップ・LP・Claim は接続と署名が必須です。

Level 2

中級者向け — 機能を使いこなす

Swap & クロスチェーンブリッジ

同一チェーンスワップ

HyperEVM 上の Router（実装コントラクト: HyperpoolRouter、デプロイ JSON キー: router）経由でプールとやり取りします。入力量の 0.30% が手数料として差し引かれ、出力量が計算されます。

計算例: 1,000 USDC → kHYPE の場合

  • 総手数料 = 1,000 × 0.30% = 3.00 USDC
  • LP 向け（プール内蓄積）= 3.00 × 86% = 2.58 USDC
  • プロトコル = 3.00 × 14% = 0.42 USDC
Li.FI クロスチェーン

Ethereum / Arbitrum / Base 等から HyperEVM へ資産をブリッジできます。Hyperpool は Li.FI API 呼び出し時 fee=0 を固定しており、追加のアグリゲーター手数料は取りません（UI: 「0% Aggregator Fees — No hidden spreads, just gas」）。

スリッページ許容は設定画面で変更可能（デフォルト 0.5% = 50 bps）。0.1% / 0.5% / 1.0% などから選択できます。

Position / LP / Liquidity Vault

方法流れ向いている人
直接 LP kHYPE + USDC を Router 経由でプールに預ける → LP トークンを受け取る 両トークンを保有している人
Zap LP 片方のトークン（kHYPE または USDC）だけを預け、半分をスワップして LP 作成 単一トークンから LP に入りたい人
Liquidity Vault Vault に預ける → Vault が LP を管理 → ユーザーに Vault シェアを付与 Vault 経由でまとめて LP 管理したい人
LP 収益は自動入金されない

手数料の 86% はプール内に蓄積されます。LP トークンを burn（流動性削除）したときに、増えたプールシェア分として受け取ります。

「上10% / 下10%」（±10% レンジ）とは？

Position 画面で ±3% / ±6% / ±10% と表示されるのは、現在価格を中心としたモニタリング用の価格帯です。「上10% / 下10%」は次の意味です。

表示意味例（現在価格 2,000 の場合）
上限（上 +10%）現在価格 × 1.102,200
下限（下 −10%）現在価格 × 0.901,800
レンジ幅合計 20%（±10%）1,800 〜 2,200

この帯は In Range / Out of Range の表示や推定 APY の参考に使われます。

重要 — オンチェーンはフルレンジ LP

±10% を選んでも、実際のオンチェーン LP は Uniswap V2 型のフルレンジです（全価格帯に流動性が有効）。表示レンジは監視用であり、集中流動性（Uniswap V3 型）は Phase 3 以降の予定です。

ポイント & 紹介プログラム

ポイントの仕組み
  1. ユーザーがスワップする
  2. 0.30% の手数料が発生
  3. PointsDistributor に手数料量と 1:1 でポイントが記録
  4. 紹介関係があればボーナス加算
  5. UI が約 5 秒ごとに残高を更新
  6. Claim ボタンで明示的に請求（自動送金ではない）
ボーナス対象内容
被紹介者ブースト 招待コードを入力した友達 獲得ポイント +10%
紹介者ボーナス コードを発行したあなた 友達が生んだポイントの +15% を追加獲得
日次エポック（Daily Pool）

エポック長は 24 時間。UI に「Daily Pool: 1,000,000 PTS」と表示されますが、これはプロダクト上の日次配布プール目標です。次のエポックへの切替は、24 時間経過後の次のスワップ手数料記録時に自動で行われます。

Cashdrop（Merkle エアドロップ）

対象者は Merkle 証明を使って USDC を請求（claim）できます。プロトコル手数料はなく、ガス代のみ必要です。ウォレット接続後、Cashdrop タブで請求可能額を確認できます。

セキュリティ注意点

  • Admin パネル（/admin）は本番ではアクセス制御を必須化
  • PRIVATE_KEY はリポジトリにコミットしない（.env / シェル環境変数のみ）
  • メインネット前: owner / feeToSetter の マルチシグ + Timelock 化を推奨
  • スポット価格（getReserves）を外部オラクルとして使わない — コールバック中は stale。価格参照は HyperpoolOracleLibrary TWAP を使用
