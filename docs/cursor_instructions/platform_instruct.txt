# 依頼の背景と目的
DeFi（分散型金融）プラットフォーム「Hyperpool (https://www.prjx.com/)」のWebアプリケーションのフロントエンド画面をモダンな技術スタックで完全再現・構築してください。
暗号資産のトレードや流動性管理を行う、極めて洗練されたクリーンかつサイバー感のあるダークモードのUI/UXを実装します。

# 技術スタックの指定
- framework: Next.js (App Router, TypeScript)
- styling: Tailwind CSS
- UI Components: shadcn/ui (またはそれに準ずるLucide Reactアイコンを用いたクリーンなTailwindコンポーネント)
- animation: Framer Motion (タブ切り替えやローディング時の滑らかな遷移用)

# 画面構成と機能要件

## 1. 全体レイアウト & 背景 (Global Layout)
- 画面全体を「ダークモード」を基調とした背景（`bg-zinc-950` や `bg-black`）に設定してください。
- ほんのりと背景にグラデーションの輝き（Radial Gradientのバックライトエフェクトなど）を入れ、サイバー・Web3感を演出してください。
- フォントは、可読性の高いSans-serif（Interなど）またはモダンな等幅フォントを組み合わせ、先進的な金融アプリの印象を持たせてください。

## 2. ヘッダー領域 (Header Component)
- 固定型（Sticky/Fixed）のヘッダーを最上部に配置してください。背景は半透明（`bg-opacity-70 backdrop-blur-md`）。
- **左側:**
  - 「Hyperpool」のロゴ、または洗練されたテキストロゴ。
  - ナビゲーションメニューとして「Swap」「Liquidity」「Portfolio」「Cashdrop」の4つのタブボタンを配置。
  - 現在選択されているアクティブなメニューには、アンダーラインや文字色の強調（ホワイト）を適用し、非アクティブはグレーアウト（`text-zinc-400`）させてください。
- **右側:**
  - ウォレット未接続時は、グラデーションや目を引くスタイルの「Connect Wallet」ボタンを表示。
  - ネットワーク切り替え用のドロップダウン（例: Hyperliquid, Ethereum等）を模したセレクター。
  - 設定用アイコン（歯車マーク）を配置。

## 3. メイン・コンテンツカード (Main Dashboard Card)
- 画面の中央に、幅 `max-w-md` (約400px〜450px) の洗練されたカード型コンポーネントを配置してください。
- カードの背景は `bg-zinc-900/80`、境界線は `border border-zinc-800` などで薄く縁取り、シャドウを効かせてください。
- ヘッダーでの選択（Swap, Liquidity, Portfolio, Cashdrop）に応じて、このメインカード内の表示内容が滑らかに切り替わるように（Framer Motion等を使って）コーディングしてください。

## 4. 各タブの内部詳細仕様

### 【A. Swap（スワップ）画面】
- **入力エリア（From）と出力エリア（To）の2段構成:**
  - それぞれに「金額入力インプット（数値のみ・枠線なしの大きなフォント）」と「トークン選択ボタン（例: USDC, HPOOL, ETHなどのロゴ＋シンボル名＋下矢印）」を配置。
  - 右側または下部に入力可能な「Max」ボタンや、現在の「Balance: 0.00」といった残高目安テキストを表示。
- **中央の入れ替えボタン:**
  - FromとToの間に、上下の矢印アイコン（↕）を配置。ホバー時に回転・拡大するインタラクションを追加。
- **価格・レート表示:**
  - 下部に小さな文字で「1 HPOOL = 0.XX USDC」といった交換レート、スリッページ設定、ネットワーク手数料（Gas）の目安を薄いグレーで表示。
- **アクションボタン:**
  - カード最下部に「Connect Wallet」または「Swap」と書かれた、横幅いっぱいの大きなプライマリボタンを配置。

### 【B. Liquidity（流動性）画面】
- 流動性プールへの参加・管理画面。
- 「+ Add Liquidity（流動性を提供する）」ボタン。
- ユーザーが現在提供しているアクティブなポジションの一覧（Your Positions）を表示するエリア（未提供時は「No active positions found.」といったクリーンなメッセージを表示）。
- プール一覧（例: HPOOL/USDC, ETH/USDC）のリストと、それぞれのAPR（年利換算）やTVL（総ロック価値）のダミー数値を美しくテーブル形式、またはカードリスト形式で配置。

### 【C. Portfolio（ポートフォリオ）画面】
- ユーザーの資産状況をダッシュボード化。
- **総資産残高:** 「Total Balance: $0.00」などの大きなタイポグラフィ。
- **内訳（Asset Allocation）:** 接続中のウォレット内トークンの内訳、預け入れ中の流動性トークン（LP）の価値を分けたカード、またはシンプルな進捗バー（Progress bar）で可視化。

### 【D. Cashdrop（キャッシュドロップ）画面】
- いわゆるエアドロップや報酬請求（Claim）の専用画面。
- 「Available Rewards」「Next Drop」などのステータスを表示。
- 報酬がある場合は「Claim」ボタンが活性化し、ない場合は「No pending cashdrops available.」と表示されるUIを構築。
- 「Loading...」と表示されるローディングインジケーターやアニメーションを各所に自然に組み込んでください。

# 実装における注意点
- 未接続状態や読み込み中を表す「Loading...」などのステータス表現を適切に入れ、リアルなWeb3アプリの挙動をモックとして体験できるようにしてください。
- すべてのコンポーネントを1つのファイル、あるいは綺麗に分割されたコンポーネントとして、そのまま動作するTypeScriptコードで出力してください。
- Tailwind CSSのクラス名は最新の記法を用い、レスポンシブ対応（スマホ〜PC）も考慮してください。

## 5. 追加タブの内部詳細仕様

### 【E. Earnings（デイリー収益ダッシュボード）画面】
ユーザーが「毎日アクセスして収益を確認・請求したくなる」ように、視覚的で動きのあるダッシュボードにしてください。

- **メイン統計（トップカード）:**
  - **「Total Lifetime Earnings（累計獲得収益）」**と、最も重要な**「Today's Estimated Earnings（本日の見込み収益）」**を大きなネオンカラー（グリーンやシアン）のフォントで強調表示してください。
  - 数字がリアルタイムでわずかにカウントアップ（Tickアニメーション）するような、ライブ感を演出するダッシュボードにしてください。
- **24時間収益グラフ（Mini Chart）:**
  - 直近7日間、または24時間の収益推移を示す、クリーンな折れ線グラフ（Rechartsなどのライブラリを模した、Tailwindによる美しいモックグラフ）を配置。
- **「Claim Daily Rewards（デイリー報酬の請求）」セクション:**
  - 今日一日で貯まった未請求の収益（例: `12.34 USDC`）を表示し、その横にホバーエフェクトの効いた**「Claim All（一括請求）」**ボタンを配置。
  - 毎日特定の時間にリセットされるカウントダウンタイマー（例: `Next distribution in 04h 22m`）を表示して、毎日の再訪問を促してください。

### 【F. Affiliate（アフィリエイト・紹介）画面】
紹介による不労所得（パッシブインカム）の状況を確認し、SNSにシェアしたくなるデザインにしてください。

- **リファラルリンク生成エリア:**
  - ユーザー固有の招待URL（例: `https://www.prjx.com/ref?code=XM79B4`）を表示するテキストボックス。
  - 右側にワンクリックでクリップボードにコピーできる「Copy Link」ボタンを配置。コピー成功時に「Copied!」とポップアップするインタラクションを付加。
- **報酬ティアと統計（3つのミニカード）:**
  - **カード1: 「Total Referred（紹介人数）」** （例: `42 Users`）
  - **カード2: 「Your Commission Rate（報酬率）」** （例: `10% Tier 2` などのレベルシステム）
  - **カード3: 「Affiliate Commissions（紹介報酬総額）」** （例: `450.00 USDC`）
- **紹介ランキング（Leaderboard）:**
  - コミュニティ内での競争意識を煽るため、上位紹介者のランキング（1位〜5位、匿名化されたウォレットアドレス `0x1a...3b` と獲得報酬）を、ゲームのスコアボードのような美しいテーブル形式で配置。

# 追記：Hyperpool の公式仕様に基づくロジックおよびUIのアップデート

先ほどの要件に加え、Hyperpoolの正確なプロダクト背景、エコシステム、およびインセンティブ設計（ポイントシステム）をUIとロジックに厳密に反映させてください。Uniswapの単なるクローンではなく、HyperEVMエコシステム初の実力派DEXとしてのブランディングをUIに落とし込みます。

## 1. ブランディングと背景情報の反映（UIテキスト・コンセプト）
- **HyperEVMエコシステムへの準拠:** 
  - ネットワーク選択UIのデフォルトを「HyperEVM」に設定してください。
  - 最も強力なプールとして「kHYPE」または「HYPE」関連のペアを最優先で表示（フィーチャー）してください。
- **チームの哲学の表現:**
  - サイトのフッターやAboutセクション（またはカードの隅）に、「100% Self-Funded. No VCs. No Seed Rounds. Built for the Community.」という思想を表現するテキストを美しく配置してください。

## 2. 「Liquidity（流動性）」および「Swap」画面のロジック強化
- **V3 AMMの手数料分配率の明記:**
  - 流動性提供（LP）の画面に、「86% of trading fees are directly allocated to LPs. Optimized for High LP Returns.」という文言を目立たせて配置してください。
- **ポイント（Points）獲得のヒント表示:**
  - LP画面およびSwap画面において、「ポイントの計算は『預けた金額』ではなく『発生させた手数料（貢献度）』に強く依存する」という旨のツールチップ、またはインフォメーションUI（⚠️マークなど）を設置してください。
  - 例：「Points are driven by fee generation, not just capital size. High volume pools yield more points.」

## 3. 「Earnings（収益）」から「Points（ポイントダッシュボード）」への仕様変更
前述の「Earnings」タブを、Hyperpoolの革新的な**「Points & Incentives」**ダッシュボードとして再設計してください。

- **デイリー100万ポイントプール（Daily 1M Points Pool）:**
  - 画面上部に「Daily Pool: 1,000,000 PTS distributed daily」と表示し、本日の配布までのカウントダウンタイマーを設置。
- **リアルタイムポイントUI（Real-time UI）:**
  - ユーザーの保有ポイント（例: `14,250 PTS`）をネオンカラーで表示し、UI上でリアルタイムに数字が微増するような視覚効果（モック）を付与。
- **最初の30日間限定：マルチプライヤー・インジケーター（1st Month Multiplier）:**
  - 「Top 100 Daily Leaderboard Multiplier (Min 1.0x active)」というバッジ、または現在の自分のデイリー順位（例: `Rank: #42 (1.5x Multiplier Active!)`）を表示するゲーミフィケーション要素を追加。

## 4. 「Affiliate（紹介機能）」の数式・ロジックの厳密化
紹介画面（リファラル）の報酬設計を、公式の相互ブースト（Dual-Reward）仕様に書き換えてください。

- **招待側の特典表示:** 
  - 「Earn 10% of all points generated by your friends.」（友達が獲得したポイントの10%を永続的に獲得）
- **被招待側（友達）の特典表示:** 
  - 「Your friends get a +10% Base Earning Efficiency Boost when using your code.」（あなたのコードを使うと、友達の基本ポイント獲得効率も10%アップする）
- この相互メリット（Win-Win構造）が明確に伝わる、2カラム、または美しいカードデザインにしてください。
- 招待コード入力欄（「Enter Invitation Code」）もPortfolioまたはAffiliate画面内に設置してください（ポイントシステム開始のトリガーとなるため）。

# Cursorへの具体的な指示タスク

## Task 1: HyperEVM専用のFoundry設定ファイル（foundry.toml）の作成
- Testnet (https://rpc.hyperliquid-testnet.xyz/evm) と Mainnet (https://rpc.hyperliquid.xyz/evm) のRPCエンドポイントおよびChain IDを明記した `foundry.toml` を出力してください。
- コントラクトサイズ制限およびデプロイ時のGas Limitエラーを回避するため、オプティマイザの設定（`optimizer = true`, `optimizer_runs = 200`）を含めてください。

## Task 2: HyperCore連携のためのインターフェース（L1Read / CoreWriter）の定義
- コントラクト側からHyperCoreの情報を取得するための、プリコンパイル用インターフェースファイル `IL1Read.sol` を作成してください。
  - アドレス: `0x0000000000000000000000000000000000000800` 付近の仕様。
  - 主な関数: `getL1BlockNumber()`, `getOraclePrice(uint256 assetId)` などを定義。
- HyperCoreへアクションを書き込むための `ICoreWriter.sol` （アドレス: `0x3333333333333333333333333333333333333333`）のインターフェースを作成してください。

## Task 3: ローカル環境（Anvil）でのテストコードとMockスクリプトの作成
- Forge（Foundry）を使用した統合テストコード `ProjectXTest.t.sol` を作成してください。
- **テストの必須アプローチ（以下のいずれか、または両方をコード化）:**
  1. **Testnet Forkモード:** `vm.createSelectFork("hyperEVM_testnet")` を使用して、テストネットの環境をローカルに複製して実行するテストロジック。
  2. **プリコンパイルのMock化（vm.etchの活用）:** ローカルの独立したAnvil環境でもテストが通るよう、`0x0000000000000000000000000000000000000800` のアドレスに対して、`vm.etch` を用いてダミーの価格データを返すMockコントラクトのバイトコードを強制注入（Deploy）するセットアップ関数（`setUp()`）を記述してください。

## Task 4: Gas Limit（2M Gas）制限を考慮したデプロイスクリプト
- HyperpoolのV3 AMMプールやファクトリー、ポイント分配コントラクトをデプロイするための、Forgeスクリプト（`DeployProjectX.s.sol`）を作成してください。
- デプロイの際、HyperEVMの「Small Block (2M Gas制限)」に引っかからないよう、一度にすべてのコントラクトをデプロイするのではなく、段階的にデプロイしてアドレスを紐付ける（あるいはProxyパターンを採用する）構成にしてください。また、Big Block（30M Gas）を利用してデプロイする際の注意点をコメントで残してください。

## Task 5: フロントエンド連携のためのローカルテスト手順書の出力
- フロントエンド開発者がローカルのAnvil（またはHardhat Node）に接続し、メタマスクのネットワークを切り替えてデバッグできるようにするための、コマンド（`anvil --fork-url ...` など）を含んだ簡易的な `README.md`（手順書）を出力してください。
